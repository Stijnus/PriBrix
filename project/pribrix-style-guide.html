14 Do / Don't
✓ DO
  • ✓ Use DM Sans exclusively for all text
  • ✓ Use red-500 (#E8192C) for all primary CTAs
  • ✓ Use r-xl (20px) radius on content cards
  • ✓ Show savings in green (#1A9E5C)
  • ✓ Use strikethrough for original prices
  • ✓ Apply red shadow shadow-red to primary buttons
  • ✓ Use uppercase + letter-spacing for category labels
  • ✓ Keep screen horizontal padding at exactly 16px
  • ✓ Use skeleton loaders (shimmer) while fetching
  • ✓ Show "BEST PRICE" badge on the cheapest retailer row
✕ DON'T
  • ✕ Never use Inter, Roboto, or system default fonts
  • ✕ Don't use purple, blue, or green as primary brand color
  • ✕ Don't use sharp corners (0px radius) on cards
  • ✕ Never show prices without currency symbol (€)
  • ✕ Don't use more than 2 font weights per screen
  • ✕ Don't use full-opacity black text on colored backgrounds
  • ✕ Never skip the non-affiliation disclaimer in settings
  • ✕ Don't use raw hex values in component styles — use theme tokens
  • ✕ Don't render more than 20 deal rows without pagination
  • ✕ Never call Supabase directly from a UI component
